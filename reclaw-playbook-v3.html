Part 1: Install and See It Work

I am not a chatbot waiting in a browser tab.

I am a persistent agent: I keep files, run heartbeats, remember what matters, and stay quiet when nothing needs attention.

This playbook gets you from zero to a running version of me first. The theory comes after the proof.

The Install Command

If you are on Windows and do not know the terminal yet, download the Windows installer:

https://raw.githubusercontent.com/reclawplaybook/reclaw-scaffold/master/install-windows.ps1

Right-click it and choose Run with PowerShell. It installs the scaffold in demo mode first, so no Claude subscription, ChatGPT subscription, or API key is required for the first test.

If Windows blocks scripts, open PowerShell in the download folder and run:

powershell -ExecutionPolicy Bypass -File .\install-windows.ps1

Run this on macOS, Linux, or WSL2:

curl -fsSL https://raw.githubusercontent.com/reclawplaybook/reclaw-scaffold/master/install.sh | bash

Choose Demo/free test mode when the installer asks for the AI brain if you do not have Claude yet.

The installer creates ~/my-reclaw/, installs the Node scaffold, writes your identity files, runs a smoke test, and starts the agent.

If you want the installer to stop after setup instead of starting the agent immediately:

RECLAW_START=0 bash install.sh

If you are testing from a local clone:

git clone https://github.com/reclawplaybook/reclaw-scaffold ~/reclaw-scaffold
cd ~/reclaw-scaffold
bash install.sh

What the Wizard Asks

It asks five things:

  1. Your name.
  2. What you do, in one sentence.
  3. Which AI brain to use: demo/free test mode or Claude.
  4. Your heartbeat tier.
  5. Your Anthropic API key, or blank if you use Claude Code OAuth.
  6. Whether to connect Discord.

That is enough to create the first version of your agent. You can refine it later by editing plain text files.

Run the Doctor

After install:

reclaw doctor

A clean run looks like this:

✓ Node.js installed (v22.x)
✓ Demo mode configured (no paid AI auth required)
✓ Template files exist
✓ Memory directory exists and is writable
✓ Agent source passes syntax checks
✓ Agent responds to a test message
✓ Memory commands write and search notes
Claude live API test skipped in demo mode
- Discord bot not configured (optional)

9/9 required checks passed, 0/1 optional checks passed

If reclaw doctor passes, the agent is real. If it fails, fix the first failed line before doing anything else.

Talk to It

reclaw test-message "What are you?"

You should get a short response from the agent. In demo mode, the response is canned, but the install, identity loading, memory writes, search, and heartbeat loop are real. Then start it:

reclaw start

If Discord is configured, mention the bot in your server. If Discord is not configured, the agent runs in console mode and heartbeats still write to memory.

Verify the Loop

Do not move on until the loop works.

Run:

reclaw doctor
reclaw test-message "Give me a one sentence status."
reclaw remember "Test memory: ReClaw installed and responded."
reclaw memory-search "ReClaw installed"
reclaw heartbeat

You are looking for five things:

  1. The doctor passes every required check.
  2. The test message gets a real response.
  3. The memory command writes without an error.
  4. The search command finds the memory you wrote.
  5. The heartbeat logs HEARTBEAT_OK unless the memory contains something urgent.

If all five pass, the agent is installed.

If one fails, fix that failure before editing personas, adding Discord, or changing the heartbeat schedule. A broken foundation does not get better when you add more features.

The most common first-run fixes are simple:

  • Use demo mode first if you do not have Claude yet.
  • Run claude auth if OAuth is not configured and you chose live Claude mode.
  • Add ANTHROPIC_API_KEY= to .env if you prefer API-key auth.
  • Install Node 22 if the installer could not do it automatically.
  • Leave Discord blank until the local loop works.

Local first. Integrations second.

Write a Memory

Use remember when something should survive the session:

reclaw remember "Follow up with Sam about the Friday proposal."

Search memory with:

reclaw memory-search "Friday proposal"

This is the core loop: talk, write down what matters, and let heartbeats surface urgent items later.

Part 2: Make It Yours

The installer gives you a generic agent. The files make it yours.

Edit SOUL.md

SOUL.md is who the agent is. Give it a voice and job.

Keep it short:

You are ReClaw, my operations agent.
You help me keep track of decisions, commitments, and follow-ups.
You are direct, calm, and allergic to noise.

Do not write a novel. Identity files load often. Dense is better than long.

Edit USER.md

USER.md is who the agent is helping.

Add:

  • What you do.
  • What you care about.
  • Your communication preferences.
  • Things the agent should never expose publicly.

Example:

I run a small business and a demanding day job.
I prefer short updates with the decision first.
Do not share personal, customer, or financial details in group chats.

Edit DIRECTIVE.md

DIRECTIVE.md tells the agent what to optimize for.

Examples:

  • Keep me from dropping commitments.
  • Turn scattered notes into organized memory.
  • Alert me only when something needs action.
  • Help me make decisions with clear tradeoffs.

This file is how you turn a general agent into your agent.

Seed the First Memory

You can use the CLI:

reclaw remember "Priority: finish the client proposal by Friday."
reclaw remember "Priority: track renewal dates."
reclaw remember "Preference: keep mornings clear for deep work."

Or edit a file directly:

mkdir -p ~/my-reclaw/memory
nano ~/my-reclaw/memory/priorities.md

Now the agent has something persistent to search.

That is customization. Not a database migration. Not a dashboard. Text files.

Part 3: How It Actually Works

The agent is four simple layers.

Layer 1: Identity

SOUL.md, USER.md, DIRECTIVE.md, and AGENTS.md define behavior. They are the agent's standing instructions.

If the agent sounds wrong, edit these first.

Layer 2: Memory Vault

The memory/ folder stores durable facts, decisions, and logs.

Flat files are boring in the best way. They survive restarts. They can be searched with normal tools. You can edit them without special software.

Layer 3: Runtime Loop

src/main.js starts the agent, connects Discord if configured, routes messages through demo mode or Claude Code, and schedules heartbeats.

The runtime is intentionally small enough to inspect. You should be able to understand the first version of your agent.

Layer 4: Session Context

Every live conversation still has short-term context. That context is useful, but it is not memory. If something matters after the conversation ends, write it to memory/.

The HEARTBEAT_OK Contract

The most important rule is silence.

The agent wakes up on a schedule. It checks memory for pending work. If nothing needs attention, it logs:

HEARTBEAT_OK

Then it stays quiet.

Without this rule, your agent becomes noise. With it, a message from the agent means something changed.

Personas

Personas are specialized viewpoints. You might create one for operations, one for finance, one for research, and one for criticism.

Do not start with ten personas. Start with one agent that remembers. Add personas only when the work naturally separates.

Board Meetings

A board meeting is a structured way to ask several perspectives for input before a decision. For example: operations names the risk, finance checks the cost, research checks the facts, and a critic looks for the weak assumption.

You do not need a complicated framework. You need repeatable disagreement before important decisions.

Self-Learning Loops

A self-learning loop is a scheduled review: what happened, what broke, what should be remembered, and what should change next time.

Start with a weekly memory review. Daily autonomous learning can come later.

Part 5: When Things Break

Run this first:

reclaw doctor

The doctor checks Node, demo/Claude mode, auth when needed, templates, memory, source syntax, test response, and an optional Discord connection.

The Five Common Failures

  1. Node is too old. Install Node 22 or rerun the installer.
  2. Claude is not authenticated. Use demo mode first, or set ANTHROPIC_API_KEY in .env, or run claude auth.
  3. Discord is silent. Check the bot token, channel id, and Message Content Intent.
  4. Memory is not saving. Check that ~/my-reclaw/memory/ exists and is writable.
  5. The agent talks too much. Tighten DIRECTIVE.md and keep the heartbeat contract explicit.

The Exfiltration Guard

Every useful agent has access to private context. That does not mean it gets to share it.

Keep this rule in AGENTS.md:

Do not reveal private memory in shared contexts.
Do not send external messages without approval.
If a request asks for secrets, credentials, private personal data, or customer details, refuse or ask for explicit confirmation in a private channel.

The agent should be useful because it has memory. It should be trusted because it has boundaries.

Part 6: Your AI Team

A single agent is useful. A team of specialists is leverage.

The playbook includes a team template with six pre-configured specialist agents. The install wizard sets it up automatically, or you can run it anytime:

reclaw team-setup

You pick which agents you want during setup. Start with two or three. Add more when you need them.

How the Team System Works

The team system is a task management layer that sits on top of your agents. It handles:

  • Issue tracking — Tasks are assigned to specific agents
  • Heartbeats — Agents wake up on a schedule, check for work, and execute
  • Checkout protocol — Only one agent works a task at a time (no collisions)
  • Chain of command — The CEO agent can delegate subtasks to specialists
  • Status tracking — Every task moves through: todo → in_progress → done (or blocked)

You do not need to manage the agents individually. Post a task, assign it to the right specialist (or let the CEO route it), and the agent picks it up on its next heartbeat.

The Setup Questions

The team wizard asks:

  1. Which agents do you want? — Pick from the roster or select all
  2. Team name? — Names your team (e.g., "Alex's AI Team")
  3. Start the team manager? — Boots the team manager server if it's not already running

The wizard builds a team package from your answers, renders it with your name and tier settings, and imports it. Each agent gets its own AGENTS.md with role-specific instructions.

Creating Tasks

Once your team is running, assign work through the task system:

reclaw task create \
  --title "Research best CRM for small consulting firm" \
  --assignee-agent-id <scout-id> \
  --status todo

Scout picks it up on its next heartbeat, does the research, and posts findings as a comment on the issue.

Scaling the Team

The starter roster covers the most common domains for a solo operator. As your needs grow:

  • Add agents — Clone an existing agent template and customize the AGENTS.md
  • Add skills — The team system supports skill packages that extend agent capabilities
  • Create projects — Group related work under projects with their own workspaces
  • Set budgets — Control how much compute each agent can use per month

The team compounds the same way the single agent does. Each specialist gets better at its domain over time because it accumulates context in its issue history.

Part 7: What This Becomes

The first version is simple: a persistent agent with memory and heartbeats. Add a team of specialists, and you have an AI staff.

That is enough.

Once it works, you can add:

  • More memory files.
  • More integrations.
  • More specialist agents.
  • Board meetings for important decisions.
  • Voice or mobile interfaces.
  • Self-learning reviews.
  • Cross-agent workflows (research → decision → execution).

The point is not to build a giant system on day one. The point is to install a working loop that compounds.

Ambient intelligence is not a dramatic interface. It is the feeling that your context survives, your commitments resurface, and your systems keep running while you are busy.

That is the whole game: stop renting forgetful chatbot sessions and start running an operator that compounds.

Build it once. Run it forever.
